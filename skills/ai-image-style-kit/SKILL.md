# 火山引擎图像工具箱

## 功能概述

| 功能 | 模型 | 说明 |
|------|------|------|
| 风格分析 | `doubao-seed-2-0-pro-260215` | 上传图片，提取风格特征并自动保存 |
| 图像生成 | `doubao-seedream-5-0-260128` | 文本描述生成图片 |

## 存储机制

风格文件保存于 `scripts/styles/style_XXX.json`，每个文件包含：
- `id`：风格编号（自动递增）
- `style_name`：风格名称（用户自定义，可为空）
- `style_description`：风格描述
- `source_image`：图片来源
- `saved_at`：保存时间

## 环境配置

**API Key 存储位置：** 参见 `TOOLS.md`（不写在代码里）
```bash
export ARK_API_KEY="你的火山引擎 ARK API Key"
```

获取方式：火山引擎控制台 → API Key 管理

## 完整工作流程

```
第1步：发图片 → 提取风格（可命名）
第2步：用户说"用xxx风格生成图片" → AI确认风格 → 引导用户输入主题 → 融合生成图片
```

## 对话式交互（推荐方式）

用户可以用自然语言描述需求，**不需要记住风格编号**，AI 自动解析。

### 提取风格
```
用户发来图片 + 可选说"提取风格"/"分析风格"/"保存为xxx风格"
```
AI 自动调用 `analyze`，返回风格编号和名称。**同时确认该风格的默认生成分辨率（如 16:9、4:3、1:1 等）。**

### 用风格生成图片（智能判断）

AI 需要判断用户说的内容是「生图要求」还是「主题描述」：

**生图要求**（需要继续询问）：用户说了图片类型但没有具体画面内容，需要继续问
- "用知识卡片风格生成公众号封面图" → 识别为「生图要求」，继续问具体内容
- "用知识卡片风格画一张海报" → 识别为「生图要求」，继续问

**主题描述**（直接提炼）：用户描述了具体画面内容，可以直接提炼
- "用知识卡片风格生成一张图：深夜办公室程序员加班" → 直接提炼
- "用知识卡片风格生成，内容是GEO垂直测评站" → 直接提炼

**判断关键词**：
- 生图要求关键词："生成"、"画一张"、"做一张"、"做一张图" + 图片类型（封面图、海报、头像等）
- 主题描述关键词：包含具体画面内容（"深夜程序员加班"、"办公室场景"等）

**示例：**
```
用户："用知识卡片风格生成公众号封面图"
AI：已选中【知识卡片】风格，默认分辨率 16:9，
    我理解你需要一张公众号封面图，请描述一下具体的画面内容？

用户："AI知识管理相关的封面图，科技感"
AI：正在提炼提示词...
    原始：AI知识管理封面图，科技感
    提炼后：AI知识管理公众号封面图，科技感配色，简洁构图，标题醒目
AI：融合风格生成中...
```

**注意**：生成图片后必须：
1. 用 `download` 命令下载到本地（URL 临时签名，需尽快下载）
2. 用 `feishu_send_image.py` 脚本发送（自动调用飞书 API 上传为真正的图片消息）

**发送命令**：
```bash
FEISHU_APP_ID=cli_a93b235b9638dcd9 FEISHU_APP_SECRET=qbaKxKfJoSNGwCfDzCG5hdm8iOpvi80N FEISHU_USER_OPEN_ID=用户的open_id python3 scripts/feishu_send_image.py <本地图片路径>
```

`FEISHU_USER_OPEN_ID` 可从会话元数据中获取（sender_id）。

凭证存储在 `TOOLS.md`，运行时自动读取。

### 第1步：提取风格（可命名）

用户发来图片时，自动提取风格并保存。支持**给风格起名字**，方便后续引用。

```
python3 scripts/doubao_image_ops.py analyze <图片路径或URL> --name=风格名称
```

**示例：**
```
# 不命名 → 系统自动分配编号 #003
analyze /path/to/image.jpg

# 起名字 → 保存为"科技感"
analyze /path/to/image.jpg --name=科技感
```

**返回示例：**
```
✅ 风格分析完成
   风格编号：#003（名称：科技感） → 已保存
```

---

### 第2步：指定风格 + 描述生成图片

用 `use` 命令加载已有风格，与文字描述融合后生成图片。

```
python3 scripts/doubao_image_ops.py use <风格编号或名称> "<生图描述>" [--size 2K] [--no-watermark]
```

**注意**：用户说"用xxx风格生成图片"时，**不要直接拼凑生成**，而是先确认风格并询问主题描述。

**支持两种引用方式：**
- **按编号**：`use 3 "<描述>"`（引用 #003）
- **按名称**：`use 科技感 "<描述>"`（模糊匹配名称）

**示例：**
```
# 用户："用科技感风格生成图片"
# AI 确认风格后询问：
#   已选中【科技感】风格，请描述你想生成的内容？
# 用户："一张科技公司官网banner"
# AI → use 科技感 "一张科技公司官网banner" → 生成

# 直接指定（底层命令）：
use 3 "一张赛博朋克风格的城市夜景海报"
use 科技感 "未来感十足的程序员头像，蓝色调"
```

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--name=名称` | 提取风格时给风格起名 | （空） |
| `--size=尺寸` | 生图尺寸，支持 1K/2K/4K、16:9/4:3/1:1、1920x1080 | **16:9** |
| `--no-watermark` | 关闭水印 | 水印开启 |
| `--format` / `-f` | 格式化输出（人类可读） | JSON输出 |

## ⚠️ 重要：风格文件只读

**风格文件（style_XXX.json）是只读的，不要修改。**

生成图片时：
- **输入**：用户提供的描述 + 已保存的风格描述
- **行为**：AI 把两者融合成生图 prompt，调用生图模型
- **禁止**：不要把用户描述写入风格文件，不要修改 style_description

### AI 提炼提示词

在融合之前，AI 先分析用户描述：
- 提取核心内容和构图意向
- 保留关键视觉元素
- 去除模糊、冗余的表达
- 输出一个清晰、精准的生图提示词

### 融合公式
```
最终 prompt = 提炼后的描述 + "，风格要求：" + style_description
```

风格文件一旦保存，内容固定不变。只有 `analyze` 命令会写入新风格。
