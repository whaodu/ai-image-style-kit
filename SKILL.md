# 火山引擎图像工具箱

## 功能概述

| 功能 | 模型 | 说明 |
|------|------|------|
| 风格分析 | `doubao-seed-2-0-pro-260215` | 上传图片，提取风格特征并自动保存 |
| 图像生成 | `doubao-seedream-5-0-260128` | 文本描述生成图片 |

## 存储机制

风格文件保存于 `scripts/styles/style_XXX.json`，由 AI 自动管理，用户无需手动操作。

**保存流程：**
1. 用户发来参考图 + 说「提取风格」
2. AI 调用 `analyze` 命令，豆包模型分析图片风格特征
3. 分析结果写入 `scripts/styles/style_XXX.json`，编号自动递增
4. AI 告知用户风格已保存，可选择命名

**提取流程（加载已有风格）：**
1. 用户说「用XX风格生成图片」
2. AI 调用 `use <名称或编号>` 命令
3. 脚本从 `scripts/styles/style_XXX.json` 读取风格描述
4. 风格描述与用户描述融合为最终 prompt，调用生图 API

**风格文件结构：**
```json
{
  "id": 1,
  "style_name": "知识卡片",
  "style_description": "极简手绘涂鸦风，低饱和莫兰迪色调...",
  "source_image": "/path/to/reference.jpg",
  "saved_at": "2026-03-24 23:13:35"
}
```

**引用方式：**
- 按编号：`use 3 "生图描述"`
- 按名称：`use 知识卡片 "生图描述"`（模糊匹配）

## 对话式交互（推荐方式）

用户可以用自然语言描述需求，**不需要记住风格编号**，AI 自动解析。

### 提取风格
```
用户发来图片 + 可选说"提取风格"/"分析风格"/"保存为xxx风格"
```
AI 自动调用 `analyze`，返回风格编号和名称。**同时确认该风格的默认生成分辨率（如 16:9、4:3、1:1 等），并告知用户可以给风格起一个易记的名字，方便后续引用。**

### 用风格生成图片（智能判断）

AI 需要判断用户说的内容是「生图要求」还是「主题描述」：

**第一步：确定风格**
- 用户**未指定风格**（如只说「生成一张封面图」）→ 先调用 `list` 列出已有风格，引导用户选择
- 用户**指定了风格**（如「用知识卡片风格」）→ 调用 `use <名称>` 匹配风格文件
- 匹配成功后，**必须先向用户确认**：「已找到【知识卡片】风格，确认使用？」用户确认后再继续

**第二步：判断是生图要求还是主题描述**

**生图要求**（需要继续询问）：用户说了图片类型但没有具体画面内容，需要继续问
- "用知识卡片风格生成公众号封面图" → 识别为「生图要求」，继续问具体内容
- "用知识卡片风格画一张海报" → 识别为「生图要求」，继续问

**主题描述**（直接提炼）：用户描述了具体画面内容，可以直接提炼
- "用知识卡片风格生成一张图：深夜办公室程序员加班" → 直接提炼
- "用知识卡片风格生成，内容是GEO垂直测评站" → 直接提炼

**判断关键词**：
- 生图要求关键词："生成"、"画一张"、"做一张"、"做一张图" + 图片类型（封面图、海报、头像等）
- 主题描述关键词：包含具体画面内容（"深夜程序员加班"、"办公室场景"等）

**示例：**
```
用户："用[风格名称]风格生成公众号封面图"
AI：已找到【[风格名称]】风格，确认使用吗？
用户："确认"
AI：请描述一下具体的画面内容？
用户："AI出海赛道分析，科技感"
AI：正在提炼提示词...
    原始：AI出海赛道分析，科技感
    提炼后：AI出海赛道分析公众号封面图，科技感配色，简洁构图，标题醒目
AI：融合风格生成中...
```

**注意**：生成图片后：
1. 下载到本地（URL 临时签名，需尽快下载）
2. （可选）如需发送到飞书对话，用 `feishu_send_image.py` 脚本上传

**发送命令**（需配置飞书环境变量 `FEISHU_APP_ID` / `FEISHU_APP_SECRET`）：
```bash
FEISHU_USER_OPEN_ID=用户的open_id python3 scripts/feishu_send_image.py <本地图片路径>
```

`FEISHU_USER_OPEN_ID` 可从会话元数据中获取（sender_id）。

> 飞书发送为可选功能，不配置则仅生成图片，不做发送。

### 纯文字生成（不用风格）
```
用户："生成一张赛博朋克风格的城市夜景"
```
AI 直接调用 `generate`。

### 第1步：提取风格（可命名）

用户发来图片时，自动提取风格并保存。支持**给风格起名字**，方便后续引用。

```
python3 scripts/doubao_image_ops.py analyze <图片路径或URL> --name=风格名称
```

**示例：**
```
# 不命名 → 系统自动分配编号 #003
analyze /path/to/image.jpg

# 起名字 → 保存为"科技感"
analyze /path/to/image.jpg --name=科技感
```

**返回示例：**
```
✅ 风格分析完成
   风格编号：#001（名称：[风格名称]） → 已保存
```

---

### 第2步：指定风格 + 描述生成图片

用 `use` 命令加载已有风格，与文字描述融合后生成图片。

```
python3 scripts/doubao_image_ops.py use <风格编号或名称> "<生图描述>" [--size 2K] [--no-watermark]
```

**注意**：用户说"用xxx风格生成图片"时，**不要直接拼凑生成**，而是先确认风格并询问主题描述。

**支持两种引用方式：**
- **按编号**：`use [编号] "<描述>"`（如 `use 3 "生图描述"`）
- **按名称**：`use [风格名称] "<描述>"`（模糊匹配名称）

**示例：**
```
# 用户："用[风格名称]风格生成图片"
# AI 确认风格后询问：
#   已选中【[风格名称]】风格，请描述你想生成的内容？
# 用户："一张科技公司官网banner"
# AI → use [风格名称] "一张科技公司官网banner" → 生成

# 直接指定（底层命令）：
use 3 "一张赛博朋克风格的城市夜景海报"
use [风格名称] "未来感十足的程序员头像，蓝色调"
```

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--name=名称` | 提取风格时给风格起名 | （空） |
| `--size=尺寸` | 生图尺寸，支持 1K/2K/4K、16:9/4:3/1:1、1920x1080 | **16:9** |
| `--no-watermark` | 关闭水印 | 水印开启 |
| `--format` / `-f` | 格式化输出（人类可读） | JSON输出 |

## ⚠️ 核心原则：风格文件只读，禁止擅自修改

**风格文件（style_XXX.json）是只读的，只有在以下情况才允许写入：**

1. **用户明确说「修改风格」** — 用户主动要求修改已有风格的描述或名称
2. **用户发图片并说「提取风格」** — analyze 命令写入新风格文件

**除此之外，任何情况下都禁止修改风格文件。**

生成图片时：
- **输入**：用户提供的描述 + 已保存的风格描述
- **行为**：AI 把两者融合成生图 prompt，调用生图模型
- **禁止**：不要把用户描述写入风格文件，不要修改 style_description

**融合公式：**
```
最终 prompt = 提炼后的描述 + "，风格要求：" + style_description
```

AI 提炼提示词：在融合之前，AI 先分析用户描述，提取核心内容和构图意向，保留关键视觉元素，去除模糊冗余表达，输出精准的生图提示词，再与风格描述融合。

风格文件一旦保存，内容固定不变。只有 `analyze` 命令会写入新风格。

## 注意事项

- 生图模型为异步API，脚本会自动轮询直到完成（约30-90秒）
- 图片默认带水印，可通过 `--no-watermark` 关闭
- 支持的生图尺寸：1K/2K/3K/4K、16:9/4:3/1:1 等比例（自动转换为像素值）、WIDTHxHEIGHT 像素尺寸（默认 16:9，即 2560×1440）
- API Base URL：https://ark.cn-beijing.volces.com/api/v3
